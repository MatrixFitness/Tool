BUILD 016 - MULTI-RUNTIME ZEROTIER RECOVERY

ZeroTier:
- Detects the currently active runtime from the running process and CLI.
- Supported homes:
  /tmp/run/zerotier
  /etc/zerotier
  /usr/local/matrix/zerotier
- Does not stop a working ZeroTier daemon.
- Prevents a second daemon from binding to port 9993.
- Stops known init services only when recovery is really required.
- Waits until port 9993 is free before starting a replacement daemon.
- Preserves each router's own identity and Node ID.
- Automatically joins the Matrix ZeroTier network when missing.
- Leaves the daemon running when ACCESS_DENIED is detected and shows the Node ID.

Update engine:
- Installs the new worker atomically into /usr/local/matrix/matrix_update_worker.sh.
- Live monitor uses short polling and automatically reconnects.
- Current and persistent update state are written.
- Success is only reported after ZeroTier validation and Matrix Doctor.
